# Часть 6: Контрольные вопросы

## 1) В каком порядке выполняются конструкторы при создании объекта подкласса?
Сначала вызывается конструктор суперкласса, а затем тело конструктора подкласса

## 2) Можно ли переопределить `static` метод? Что такое method hiding?
Переопределить нельзя, но можно скрыть с методом hiding

## 3) Что произойдет, если в `switch` нет `default` и ни один `case` не совпал?
Ничего не произойдёт и выполнение продолжится со строчки после switch

## 4) Почему в `for-each` нельзя удалять элементы коллекции? Как правильно?
В for-each используется внутренний итератор что не позволяет удаление элементов во время обхода. Это приведёт к ConcurrentModificationException
Как вариант можно использовать оператор явно через .iterator() и удалять через .remove()

## 5) Чем отличается `break` с меткой от обычного `break`?
Обычный break завершает только ближайший цикл, switch
break с меткой завершает выполнение блока, помеченного этой меткой

## 6) Что означает `@Override`?
@Override указывает, что метод переопределяет метод суперкласса или реализует метод интерфейса

## 7) Какой модификатор доступа выбрать для видимости в подклассах?
Для видимости только внутри пакета  package-private (без модификатора)
Для видимости в подклассах protected

## 8) Разница между `final` классом и `sealed` классом?
final класс нельзя наследовать вообще
sealed класс можно наследовать, но только определёнными классами

## 9) Можно ли иметь несколько конструкторов?
Да, конструкторы могут отличаться набором параметров

class Point {
    Point() { ... }
    Point(int x) { ... }
    Point(int x, int y) { ... }
}
## 10) Что такое ковариантный тип возвращаемого значения?
Возможность в переопределённом методе возвращать не тот же тип, что в суперклассе, а его подтип.
